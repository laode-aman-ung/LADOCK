Documentation

Complete guide to installing, configuring, and using LADOCK Desktop for molecular docking workflows.

📋 Requirements

LADOCK Desktop requires the following software on your system before installation:

DependencyMinimum VersionNotes
Python3.103.11 or 3.12 recommended
PySide66.5Qt6 Python bindings (GUI)
NumPy1.24Array computations
SciPy1.10Scientific utilities
pandas2.0Result data handling
RDKitanyOptional — SMILES rendering & ligand prep
💡
Tip: Using Miniconda or Anaconda is the easiest way to manage all dependencies, especially RDKit which can be tricky to install with pip alone.

⬇️ Download installers

Each version ships a separate installer bundling only the engines it needs — no need to download 800 MB of binaries for a platform you don't use. Free for non-commercial use until 2030.

⬇️ Install from source

Windows

Open PowerShell or Command Prompt and run:

# Clone the repository
git clone https://github.com/laode-aman-ung/LADOCK.git
cd ladock-desktop

# Install core dependencies
python -m pip install -e .

# Optional: install with RDKit support
python -m pip install -e ".[rdkit]"

Linux / macOS

git clone https://github.com/laode-aman-ung/LADOCK.git
cd ladock-desktop
pip install -e .

Using Conda (recommended)

# Create a dedicated environment
conda create -n ladock python=3.11
conda activate ladock

# Install RDKit via conda (easier than pip)
conda install -c conda-forge rdkit

# Install LADOCK
pip install -e .
ℹ️
Bundled binaries: AutoDock Vina, AutoDock 4, ADFRsuite, MGLTools, and OpenBabel are included in the bin/ directory. You do not need to install these separately.

🚀 Launching LADOCK

After installation, start LADOCK using the launcher appropriate for your platform:

Windows

# Double-click in File Explorer, or run from terminal:
ladock.bat

Windows with WSL backend

ladock-wsl.bat

Linux / macOS

bash ladock.sh

Cross-platform (Python)

python main.py
⚠️
First launch: LADOCK may take a few seconds on first launch as it detects bundled tool paths. Subsequent launches are faster.

📖 Usage Overview

LADOCK organizes your work into Projects. A project contains your receptor, ligand library, docking configuration, and all results in one structured directory. The main window is divided into panels:

  • Docking Prep — load receptor and ligands, set search box
  • Job Manager — configure and launch batch docking
  • Result Explorer — browse scores, poses, and interaction maps
  • 3D Viewer — interactive molecular visualization
  • Ligand Library — manage and filter your compound set

🧪 Preparing Input

Loading a receptor

Go to Docking Prep → Receptor and click Load Receptor. Accepted formats:

  • .pdb — LADOCK converts to PDBQT automatically via ADFRsuite
  • .pdbqt — loaded directly

After loading, LADOCK will attempt to add polar hydrogens and assign Gasteiger charges. Review the log output in the panel for any warnings.

Loading ligands

Go to Ligand Library → Import. Supported formats:

  • .sdf — multi-molecule SD file
  • .csv — with a SMILES column (column header: smiles or SMILES)
  • .pdbqt — pre-prepared ligands (no conversion needed)
💡
If RDKit is installed, LADOCK will render 2D SMILES depictions in the ligand table. Without RDKit, ligands are still functional but displayed as text only.

⚙️ Configuring Docking

Setting the search box

In the Docking Prep panel, set the docking box center and dimensions:

  • Enter X/Y/Z center coordinates manually, or
  • Click a residue in the 3D viewer to center the box on that position
  • Adjust box size (in Å) to cover the binding pocket

Choosing a docking engine

Select the engine from the dropdown in the Job Manager panel:

  • AutoDock Vina — best default choice; fast and accurate
  • AutoDock 4 — classical method; good for benchmarking
  • VinaGPU — GPU-accelerated; requires external binary (see GPU Engines)
  • AutoDock-GPU — massive throughput on NVIDIA GPUs

Key parameters

ParameterDefaultDescription
exhaustiveness8Search thoroughness (higher = slower but more accurate)
num_modes9Number of binding poses to output per ligand
energy_range3Max energy difference from best pose (kcal/mol)
cpuautoNumber of CPU threads to use

▶️ Running a Docking Job

  1. Ensure your receptor and at least one ligand are loaded.
  2. Verify the search box covers your target binding site.
  3. Select the docking engine and set parameters in Job Manager.
  4. Click Run Docking (or Run Batch for multiple ligands).
  5. Monitor progress in the job queue — each ligand shows its status (queued / running / done / failed).
  6. When all jobs complete, results are automatically loaded into the Result Explorer.
ℹ️
Jobs can be paused or cancelled at any time from the Job Manager. Completed results are preserved even if you cancel the remaining queue.

📊 Viewing Results

The Result Explorer shows a sortable table of all completed docking runs with columns:

  • Ligand — ligand name or identifier
  • Best Score — lowest binding energy (kcal/mol)
  • Poses — number of poses returned
  • Engine — which docking engine was used

Click any row to:

  • Load the docked pose in the 3D Viewer
  • View the interaction analysis panel (H-bonds, hydrophobic contacts, etc.)
  • Cycle through all poses using the pose navigator

Exporting results

Use File → Export Results to save the current result table as .csv or .xlsx. Individual poses can be exported as .pdbqt or .pdb.

🖥️ Platform Support

LADOCK Desktop runs natively on Windows and Linux. Each platform has a dedicated launcher and installer.

🪟 Windows (native)

Run LADOCK directly on Windows — no Linux or WSL required for CPU docking.

  1. Clone the repository and run install.bat — it detects your Python (Miniconda / Anaconda / system) and installs all dependencies automatically
  2. A shortcut LADOCK.bat is created on your Desktop
  3. Or launch manually: ladock.bat
💡
The 3D molecular viewer runs fully inside the app on Windows (Qt WebEngine + 3Dmol.js). No browser needed.

🐧 Linux (native)

Run LADOCK directly on any modern Linux distribution.

  1. Clone the repository and run bash install.sh — it finds Python in conda, venv, pyenv, or your system and installs all dependencies
  2. A .desktop shortcut is created in ~/.local/share/applications
  3. Or launch manually: bash ladock.sh
💡
The 3D viewer runs in-app on Linux when PySide6-WebEngine is available. If it opens in your browser instead, install the extra: pip install -e ".[webengine]"
ℹ️
2D ligand diagrams (LID style) require RDKit: pip install -e ".[rdkit]" or conda install -c conda-forge rdkit. All other features work without RDKit.

🔗 Windows + WSL (Linux binaries on Windows)

To run GPU-accelerated Linux docking engines (VinaGPU, AutoDock-GPU) from Windows, enable WSL 2:

  1. Open PowerShell as Administrator and run wsl --install
  2. Install a Linux distribution (e.g., Ubuntu 22.04) from the Microsoft Store
  3. Inside your WSL distribution, install the GPU docking binaries
  4. Launch LADOCK using ladock-wsl.bat — it auto-detects your WSL distribution
  5. In LADOCK, go to Settings → Tool Paths and set the VinaGPU / AutoDock-GPU binary paths
⚠️
WSL GPU passthrough requires WSL 2 (not WSL 1) and an NVIDIA GPU with Windows drivers that include CUDA support via WSL.

⚡ GPU Engines

VinaGPU and AutoDock-GPU are not bundled — they must be installed separately and configured in LADOCK's settings.

VinaGPU

A GPU-accelerated version of AutoDock Vina. Install inside your WSL environment or natively on Linux, then add the binary path in Settings → Tool Paths → VinaGPU.

AutoDock-GPU

Provides massive throughput on NVIDIA GPUs by running thousands of docking evaluations per second. Best suited for large-scale virtual screening. Configure via Settings → Tool Paths → AutoDock-GPU.

💡
For typical drug discovery projects with hundreds of ligands, AutoDock Vina (CPU) is often fast enough without GPU setup. GPU engines shine with library sizes in the tens of thousands.
